# portal-site-resource

## 项目简介

portal-site-resource 是一个用于归档和发布多个独立 HTML 页面的资源仓库。本仓库不针对任何特定域名或网站，仅作为页面资源的集中存放与版本管理场所，便于后续访问、维护与分发。

## 目录结构

```
portal-site-resource/
├── pages/          # 独立 HTML 页面文件
├── assets/         # 静态资源文件（CSS、JS、图片等）
├── docs/           # 文档与说明文件
└── README.md       # 本文件
```

- **pages/**：存放所有归档的独立 HTML 页面，每个页面以独立文件或子目录形式存在。
- **assets/**：存放页面所需的公共或专属静态资源，按需组织。
- **docs/**：存放与页面内容相关的说明文档或使用指南。

## 页面归档说明

本仓库中的页面均为独立、自包含的 HTML 文件，不依赖外部特定服务或数据库。页面内容可能涉及信息展示、工具页面或静态交互示例。

- 每个页面文件或文件夹在 `pages/` 下拥有独立命名空间，以避免冲突。
- 页面内引用的资源（CSS、JavaScript、图片等）优先使用相对路径，并存放于 `assets/` 对应子目录下。
- 归档时请保持页面结构完整，确保在本地或静态服务器环境下可直接访问。

## 维护说明

- **新增页面**：在 `pages/` 下创建新文件夹，将 HTML 文件及依赖资源一并放入，并在 `assets/` 中更新公共资源（如有）。
- **更新页面**：直接修改对应文件，并确保资源路径与引用关系保持一致。
- **删除页面**：移除 `pages/` 下对应文件夹及 `assets/` 中不再使用的资源。
- **版本记录**：建议通过 Git 提交记录追踪每次变更，必要时在 `docs/` 中维护变更日志。

## 使用方式

本仓库可直接克隆到本地，或通过静态托管服务部署。所有页面均为纯静态，无需后端环境即可运行。

```bash
git clone https://github.com/your-username/portal-site-resource.git
cd portal-site-resource
```

打开 `pages/` 下的任意 HTML 文件即可查看内容。
